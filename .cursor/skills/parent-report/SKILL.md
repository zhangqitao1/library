# 家长周报 / 学习复盘

> 把分散在 `学情/` 的数据聚合成"家长 3 分钟读完"的反馈。**面向家长，不是面向小雨**：客观、具体、给可执行建议，重点突出"哪进步了、哪还卡着、家长能帮什么"。

## 数据来源（按需读）

| 看什么 | 读哪 |
|---|---|
| 她最近问了什么、关心什么 | `学情/提问记录.jsonl`（用法见 `.cursor/rules/提问追踪.mdc`） |
| 做了哪些复习/讲题、表现如何 | `学情/复习日志.md` |
| 哪些错题到期没清、反复错 | `学情/待重做清单.md` + 各册 `错题本/` |
| 各科掌握度与薄弱点 | `学情/小雨-学情档案.md` + 各册 `00-册地图.md` |

## 步骤

1. **定时间范围**：默认"最近 7 天"；家长指定就按指定（如"这个月""期末以来"）。
2. **按时间过滤聚合**：
   - 提问记录：按科目/主题归类，找**高频、反复出现**的问题 → 往往是兴趣点或真实薄弱区。
   - 复习日志：本期做了哪些科目/范围，亮点与卡点。
   - 待重做清单：到期未清的错题、被顺延（=反复做错）的题。
3. **生成报告**（见下方结构，控制在一屏内，别堆数据）。
4. **回填学情**（重要）：从本期数据里**新发现**的兴趣/薄弱点，更新 `学情/小雨-学情档案.md` 的"基本情况/各科速览/薄弱点"——让画像越来越准。

## 报告结构（简短、具体、可执行）

```
# 小雨学习周报（YYYY-MM-DD ~ YYYY-MM-DD）

## 一句话总览
<本期整体状态：在学什么、状态如何>

## 亮点（夸过程）
- <具体进步，如"政权并立表能自己复述了"——夸努力/策略，不夸聪明>

## 还卡着的点（优先级从高到低）
- <科目·知识点>：<怎么卡的 / 反复错几次>

## 她最近关心 / 爱问什么
- <从提问记录提炼，可反映兴趣，家长可顺势引导>

## 给家长的 1~2 个具体建议
- <可落地的小动作，如"周末陪她把生地思维导图合书讲一遍">

## 到期该回炉的错题
- <待重做清单里到期未清的，提醒安排>
```

## 边界
- 这是给家长的**本地**报告，数据来自 `学情/`（不上网）。**不要**把它写进会公开发布的 `subjects/`。
- 只基于真实记录说话；数据少时如实说"本期记录较少"，**不编造**进步或问题。
- 语气客观、建设性，避免制造焦虑；多用成长型措辞（夸努力/策略）。
